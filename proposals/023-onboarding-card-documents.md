# Proposal 023: 온보딩 카드 문서화와 기본 카드 설계

## 배경

실제 플레이 가능한 카드풀을 만들기 전에, 온보딩용 기본 카드부터 정리한다.

사용자는 각 카드마다 별도 문서가 필요하다고 보았다. 이번 작업은 카드 효과, 수량, 범위 표기, 사람 아이콘 위치, 매복/보류/출정 같은 키워드를 잃어버리지 않도록 카드별 문서 구조를 먼저 만든다.

## 용어 메모

확정 또는 임시 용어:

- `병력`: 카드 오른쪽 위 검은 원에 적는 수치이자, 카드 아래에 놓이는 점령/생존/소모 단위. 기존 `역량`과 `힘` 표현은 모두 `병력`으로 통일한다.
- `보류`: 군영의 카드를 뒷면으로 놓고 다음 보급 때 그 수만큼 추가 보급을 받는 행위. 패 최대 장수 10장 때문에 보류로 챙길 수 있는 장수는 제한된다.
- `매복`: 뒤집어서 지형에 둘 수 있는 카드.
- `전투`: 하나의 유닛과 유닛 간에 일어나는 한 번의 싸움.
- `교전`: 전투 중인 병사/장군들이 서로의 공격범위에 엮여 만들어진 클러스터. 교전은 별도 페이즈가 아니라 전장 위 상태다.

### 폐기된 용어 후보

`역량` 대체 후보였던 것:

- `군세`: 카드 아래에 놓이는 병력/생명/점령 여력을 모두 담기 좋다.
- `전력`: 전투와 점령에 모두 쓰이는 수치로 읽힌다.
- `위세`: 장군/세력 카드 톤에는 좋지만 기본 병사 수치로는 약간 추상적이다.
- `병력`: 목숨과 소모치를 직관적으로 설명하지만, 점령 카드 아래에 놓이는 구조까지는 약간 좁다.

확정 방향은 `병력`이다.

`힘` 대체 후보였던 것:

- `병력`: 하나씩 줄고 늘어나는 단위로 가장 직관적이다.
- `체력`: 목숨처럼 읽히지만 점령/소모 비용에는 덜 맞는다.
- `사기`: 소모/회복은 자연스럽지만 병사 수를 나타내기에는 추상적이다.
- `전력`: 상위 수치로 쓰면 좋지만 개별 소모치로는 `군세`와 역할이 겹친다.

확정 방향은 `힘`도 `병력`으로 통일하는 것이다. 예: “병력 2인 카드는 카드 아래에 병력 2개를 놓고 시작한다.”

## 전투와 교전

전투와 교전은 다르다.

- `전투`: 하나의 유닛과 유닛 사이에서 일어나는 한 번의 싸움.
- `교전`: 전투 중인 병사 또는 장군들이 상대의 공격범위에 엮인 클러스터.
- 교전은 예전처럼 별도의 페이즈가 아니라, 전장 위에서 형성되는 상태다.

교전이 끝나는 조건:

1. 교전에 엮인 병사 또는 장군이 상대의 공격범위 밖으로 벗어난다. 이 경우는 퇴각으로 본다.
2. 해당 교전에 엮인 한쪽 병사 또는 장군들이 전멸한다.

교전 종료 보상:

- 승리한 플레이어는 해당 교전에서 이긴 병사 수만큼 `점령` 포인트를 얻는다.
- 얻은 점령 포인트만큼 즉시 점령카드를 배치할 수 있다.

## 문서화 방향

승인 후 다음 위치에 카드별 문서를 만든다.

```text
docs/cards/onboarding/
  neutral-soldiers/
  neutral-tools/
  neutral-operations/
  yemaek/
  saro/
  garak/
  sipje/
```

각 카드 문서에는 다음 필드를 둔다.

- 카드명
- 카드 종류
- 진영/클래스
- 수량
- 장악범위
- 장악범위 사람 아이콘 위치
- 공격범위
- 공격범위 사람 아이콘 위치
- 병력
- 타입/키워드
- 효과
- 온보딩 역할
- 확인 필요 사항

## 확인 질문

1. `사람아이콘 = 2/5/8`은 카드 일러스트 위 아이콘 배치 좌표인지, 범위 기준점인지 확정이 필요하다.
2. `보류 없이 공격`의 의미는 보류 비용 없이 공격한다는 뜻인지, 원래 공격을 위해 보류해야 하는 별도 비용 규칙이 생긴다는 뜻인지 확인이 필요하다.
3. 도구의 `이 지형에 병사가 도달`은 이동으로 들어올 때만인지, 출정/재배치로 놓일 때도 포함하는지 확인이 필요하다.
4. 장군 카드 4장은 `캐릭터 특성 병사` 섹션에 섞어 둘지, `캐릭터 특성 장군` 섹션으로 분리할지 확정이 필요하다.

## 제안 작업 범위

사용자가 `go`라고 말하면:

1. `task.csv`에 새 작업을 추가한다.
2. `docs/cards/onboarding/` 아래에 카드별 문서를 만든다.
3. 공통 카드 문서 템플릿을 만든다.
4. 중립 병사, 중립 도구, 중립 작전 문서, 예맥, 사로국, 가락, 십제 섹션을 나눠 정리한다.
5. `docs/rules/v0.1.md`에 새 키워드 `매복`, `보류`, 패 최대 장수 10장, 숫자 키패드 범위 표기법, 전투/교전의 차이를 반영한다.
6. 기존 `역량`/`힘` 표현을 `병력`으로 일괄 정리한다.
