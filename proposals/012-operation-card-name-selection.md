# Proposal 012: 작전 카드명 선별 교체

## 배경

현재 카드 DB의 작전 카드 일부 이름이 병사/장군/유닛처럼 읽힌다. 예를 들어 `작은 호랑이` 같은 이름은 교전 중 사용하는 작전 카드명보다는 캐릭터명에 가깝다.

## 목표

- 기존 `grns-card` 카드풀에서 작전처럼 보이는 이름만 골라 현재 작전 카드 10종에 적용한다.
- 카드명은 행동, 명령, 전술, 사건, 교전 결과처럼 읽히게 한다.
- 기존 원칙인 카드풀 재사용과 시리얼 컨벤션을 유지한다.

## 선별 기준

작전 카드명 후보로 우선 채택:

- 명령형/행동형 이름: `퇴각`, `돌파`, `습격`, `추격`, `정비`, `증원`처럼 행동을 떠올리게 하는 이름
- 전술/전장 상황 이름: `매복`, `포위`, `혼란`, `전열`, `보급`, `기습`처럼 교전 중 사건으로 읽히는 이름
- 효과와 함께 읽었을 때 작전 카드처럼 자연스러운 이름

제외:

- 특정 인물, 동물, 괴이 개체처럼 보이는 이름
- 병사 카드명처럼 주체가 되는 이름
- 장군/성주처럼 보이는 이름

## 적용 범위

- `src/data/cards.json`의 `operations` 10종
- 필요하면 `sourceSerial`, `sourcePackId`, `serial`, `sigil`, `flavor`도 함께 원본 후보에 맞춘다.

## 확인 질문

1. 작전 카드의 `effect`는 일단 그대로 두고 이름/원본 시리얼만 바꿀까?
2. 후보가 10장보다 많으면 `퇴각`, `증원`, `급습`, `방어`, `처치`처럼 현재 교전 룰 키워드와 가까운 이름을 우선할까?
3. 작전 카드도 기존처럼 `tu01`, `ob01`, `ex01`, `st01` 등 모든 팩에서 고를까, 아니면 기본팩/확장팩 같은 특정 범위로 제한할까?

## 실행 조건

사용자가 `go`라고 말하면 `task.csv`에 작업을 추가하고, 기존 카드풀에서 후보를 뽑아 작전 카드 10종의 이름과 관련 메타데이터를 교체한다.
