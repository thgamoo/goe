# Proposal 011: 프로젝트 진행상황 Sync 문서

## 배경

다른 컴퓨터와 Obsidian 기반 LLM wiki에서 `괴이전` 작업 흐름을 이어보기 위해, 현재 진행상황을 Markdown 문서로 정리한다.

## 목표

- 지금까지 결정된 게임/웹사이트/카드 DB 작업 내용을 한 문서에서 파악할 수 있게 한다.
- 다른 LLM이 이어서 작업할 때 필요한 맥락을 남긴다.
- 현재 남은 의사결정과 다음 작업 후보를 분리해서 기록한다.

## 제안 위치

- `docs/project-sync.md`

## 제안 구성

1. 프로젝트 개요
2. 저장소/배포/개발 환경
3. 현재 웹사이트 구조
4. 현재 룰 결정사항
5. 카드 DB와 카드 프레임 현황
6. 확정된 작업 규칙
7. 최근 결정 및 미해결 쟁점
8. 다음 작업 후보
9. 다른 LLM에게 전달할 주의사항

## 포함할 핵심 내용

- 게임명은 `괴이전`.
- Vite base/context path는 `/goe/`.
- GitHub Pages는 push 시 배포되도록 workflow가 있다.
- 카드 DB는 JSON으로 관리하고, 현재 병사 15종, 장군 2종, 작전 10종 예시가 있다.
- 카드명/시리얼은 기존 `grns-card` 카드풀을 재사용 중이다.
- 시리얼은 `st01-r001`처럼 `팩-레어도+세자리번호` 형식을 쓴다.
- 카드 프레임은 기존 병사 카드 프레임을 사용하되, 장군 카드는 별도 가로 카드 레이아웃을 쓴다.
- 작전 카드는 무채색 처리한다.
- 카드 위치 조정은 CSS 파일이 아니라 `src/App.tsx`의 `CardPreview` Tailwind className에서 한다.
- 작전 카드명은 현재 일부가 병사/유닛명처럼 보여서, 작전다운 이름으로 교체해야 한다.

## 확인 질문

1. sync 문서는 `docs/project-sync.md`에 두면 될까, 아니면 Obsidian에서 바로 물기 좋게 `wiki/`나 `notes/` 같은 별도 폴더를 만들까?
2. 문서 톤은 다른 LLM이 바로 이어 작업하기 좋은 “작업 지시/컨텍스트” 형태로 쓸까, 사람이 읽기 좋은 “개발 일지” 형태로 쓸까?
3. 카드/룰/웹사이트를 한 파일에 모을까, 아니면 `project-sync.md`, `rules-sync.md`, `card-db-sync.md`처럼 나눌까?

## 실행 조건

사용자가 `go`라고 말하면 위 구조로 실제 sync 문서를 작성하고 `task.csv`에 작업을 기록한다.
