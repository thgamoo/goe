# Proposal 017: 교전 페이즈 제거와 차례 중 카드 사용 규칙 정리

## 배경

현재 룰북 `docs/rules/v0.1.md`는 전투가 발생하면 별도의 `교전` 절차로 들어가고, 교전이 끝날 때마다 국면이 전환되는 구조다.

사용자는 게임 흐름이 루즈해질 가능성을 줄이기 위해 `교전 페이즈` 자체를 없애고, 네 종류 카드 중 `장군`을 제외한 `병사`, `도구`, `작전` 카드를 본인의 차례에 낼 수 있는 구조로 바꾸기로 했다.

## 목표

- `교전`을 독립 페이즈/절차로 다루는 규칙을 제거한다.
- 본인 차례 중 사용할 수 있는 카드 종류를 `병사`, `도구`, `작전`으로 정리한다.
- `장군`은 기존처럼 시작 시 공개하는 고유 카드로 남기고, 일반적인 차례 중 사용 카드에서 제외한다.
- 룰북 안에서 새 구조와 상충되는 문장을 찾아 수정한다.
- `훈련소`, `훈련`, `징집` 계열 용어를 `기지`, `보급` 중심으로 바꾼다.

## 제안 규칙 방향

1. 차례 중 카드 사용
   - 플레이어는 자기 차례에 `병사`, `도구`, `작전` 카드를 낼 수 있다.
   - 차례당 카드 사용 제한은 현재 패의 장수다.
   - 이번 차례에 사용하지 않을 카드는 뒤집어 놓을 수 있다.
   - 뒤집어 놓은 카드는 다음 차례에 그 수만큼 추가 보급을 받게 해준다.
   - 뒤집어 놓았던 카드는 정해진 순서대로 기지 아래로 돌아간다.
   - `장군` 카드는 차례 중 내는 카드가 아니라, 게임 시작 시 공개하고 지속 능력 또는 별도 능력으로 다룬다.

2. 병사 카드
   - 기존 `훈련`, `징집`, `훈련소` 표현은 새 흐름과 맞지 않으므로 제거한다.
   - 병사, 도구, 작전이 모두 같은 공급 흐름에서 오도록 `기지`에서 `보급`을 받는 표현으로 바꾼다.
   - 병사는 자기 차례에 조건을 만족하면 전장에 낼 수 있다.
   - 병사 배치 가능 위치는 기존처럼 자기 영향권을 우선 기준으로 유지한다.
   - 병사는 낸 차례에 바로 이동하거나 공격할 수 없다.

3. 도구 카드
   - 현재 룰북에 별도 `도구` 카드 섹션이 없으므로 새 카드 종류로 추가한다.
   - 도구는 병사에게 장착하거나 지형에 내려놓을 수 있다.

4. 작전 카드
   - 기존에는 `교전 중 사용`으로 정의되어 있으나, 새 규칙에서는 자기 차례에 사용하는 카드로 바꾼다.
   - 작전은 상대 차례에 반응으로 사용할 수 있는 카드도 둔다.
   - 작전의 정식 표현은 `작전 문서`로 한다.
   - 일상 지칭은 `작전`으로 둔다.
   - 반응 작전은 `공격 선언 후` 또는 `상대 작전 선언 후` 사용할 수 있다.

5. 전투 처리
   - `교전`이라는 별도 페이즈는 제거한다.
   - 전투는 차례 중 행동 하나로 처리한다.
   - 공격 선언, 피해 적용, 처치, 점령 변화는 즉시 해결되는 절차로 축소한다.
   - 기존 `전멸`, `항복`, `퇴각`, `증원`, `작전 카드 5장/4장 드로우`, `역량으로 작전 비용 지불` 규칙은 삭제하거나 새 카드 효과 문법으로 재작성한다.

6. 국면 전환
   - 국면 개념은 유지한다.
   - 한 차례 전투가 끝나면 국면이 전환된다.
   - 국면 전환 시 날씨가 바뀐다.
   - 날씨 카드는 추후 별도 가이드에 따라 정리한다.
   - 초기 방향은 각 플레이어가 날씨 카드 3장씩 골라 넣는 구조다.

## 보급 흐름 초안

1. 자기 차례 시작 시 기지에서 카드 4장을 보급받는다.
2. 이전 차례에 사용하지 않고 뒤집어 놓은 카드가 있다면, 그 장수만큼 추가 보급을 받는다.
3. 이번 차례에 패에서 원하는 만큼 카드를 사용할 수 있다.
4. 사용하지 않을 카드는 제한 없이 뒤집어 놓고 다음 차례의 추가 보급 수를 만든다.
5. 뒤집어 놓았던 카드는 다음 보급 처리 후 정해진 순서대로 기지 아래로 돌아간다.
6. 뒤집어 놓은 카드는 상대에게 장수만 보이고 카드 종류와 내용은 공개하지 않는다.

## 작전 카드 용어 후보

- `작전`: 가장 짧고 직관적이다. 카드 종류명으로 쓰기 좋고, 실제 플레이 중 말하기 쉽다.
- `작전 문서`: 확정된 정식 명칭이다. 실제 플레이 중에는 흔히 `작전`으로 줄여 부른다.
- `작전서`: `작전 문서`보다 짧고 카드명/문서 느낌이 모두 난다. 정식 명칭으로 가장 무난한 후보.
- `전술`: 카드가 즉시 효과나 반응 효과 중심이라면 자연스럽다. 다만 `작전`보다 규모가 작게 느껴진다.
- `전령`: 사람이 전달하는 존재처럼 읽혀 카드 종류명으로는 약간 캐릭터 카드와 섞일 수 있다. 특정 반응 카드명이나 키워드로는 좋다.
- `군령`: 명령서 느낌이 강하고 시대극 톤에 잘 맞는다. 다만 모든 작전 효과가 명령으로 읽혀야 자연스럽다.
- `비책`: 숨겨둔 수, 반응 카드, 기습 효과에 잘 맞는다. 전체 작전 카드 종류명으로 쓰면 약간 비밀 카드 쪽으로 의미가 기운다.

확정 방향은 정식 명칭 `작전 문서`, 일반 지칭 `작전`이다. 예: “작전 문서는 보통 작전이라고 부른다.”

## 수정 대상

- `docs/rules/v0.1.md`
  - 게임 개요의 기존 전략/교전 기반 설명.
  - 주요 용어의 `국면`, `교전`, `증원`.
  - 진행 방식의 `교전이 끝날 때마다 국면 전환`.
  - 기본 명령의 `전투` 설명.
  - 카드 구성의 `작전 카드는 교전 중 사용한다`, `역량으로 작전 비용 지불`.
  - 전투 절차와 `## 교전` 전체 섹션.
  - 전멸/항복/퇴각 섹션.
  - 미정리 항목 중 교전 관련 항목.

- `src/App.tsx`
  - 룰 탭 요약 문구 중 `교전`, `국면 전환`, `작전 카드로 전멸/항복/퇴각` 관련 문장.
  - 기본 명령 설명 중 `전투`가 `교전을 시작한다`고 말하는 문장.

- `src/data/cards.json`
  - 작전 카드의 `timing: "교전"`.
  - 카드 효과 중 `교전 중`, `교전 시작 시`, `다음 교전`, `참여 병사`, `퇴각`, `항복`, `증원` 전제를 가진 문장.

- `docs/project-sync.md`
  - 현재 룰 요약과 작전 카드명/역할 설명 중 교전 기반 문장.

## 확인이 필요한 결정

1. 기존 `역량` 수치는 교전 비용 자원에서 어떤 역할로 바꿀까?
   - 전투력.
   - 행동 비용.
   - 카드 사용 제한.
   - 삭제 또는 다른 능력치로 변경.

2. 날씨 카드는 각 플레이어가 3장씩 넣어 만드는 공동 날씨 흐름으로 보인다. 섞는 방식, 공개 방식, 날씨 지속 기간은 추후 날씨 가이드에서 정한다.

3. `국면마다 1회` 효과는 그대로 유지하되, 전투 종료로 국면이 전환될 때마다 다시 사용할 수 있게 할까?

## 진행 방식

사용자가 `go`라고 말하면:

1. `task.csv`에 새 작업을 추가한다.
2. `docs/rules/v0.1.md`를 교전 페이즈 없는 차례 중심 규칙으로 수정한다.
3. 앱의 룰 탭 문구를 새 규칙에 맞게 수정한다.
4. 카드 DB의 명백한 `교전` 타이밍 표기를 새 임시 타이밍으로 바꾸거나, 효과 재작성 필요 항목을 표시한다.
5. `docs/project-sync.md`에 현재 룰 변경 사항을 반영한다.
6. `pnpm` 기반 검증 명령을 실행해 문서/앱 변경이 깨지지 않는지 확인한다.
