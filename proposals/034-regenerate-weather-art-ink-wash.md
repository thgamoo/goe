# Proposal 034: 날씨 카드 수묵화 톤 재생성

## 배경

현재 날씨 카드 일러스트는 카드별 색조가 강해 전체 카드 프레임과 어울리는 수묵화 톤에서 벗어난다.

이번 작업은 날씨 카드 이미지를 검은색과 흰색의 농담 중심으로 다시 생성하여, 괴이전의 어둡고 거친 수묵 카드 톤에 맞추는 것이다.

## 기본 방향

- 흑백 수묵화, 먹 번짐, 거친 붓질, 한지 질감
- 검은색과 흰색의 구분이 명확해야 한다.
- 채도 높은 색은 피하고, 기본은 흑백으로 둔다.
- 사용자가 제공한 예시처럼 한지 여백 위에 강한 검은 실루엣, 번지는 먹, 거친 가장자리를 사용한다.
- 카드마다 필요한 경우에도 포인트 색은 아주 작고 어둡게 제한한다.
- 카드 프레임과 텍스트가 얹히므로 테두리, 글자, 로고, 워터마크는 넣지 않는다.
- 하단 20%는 텍스트 영역을 위해 너무 복잡하지 않게 둔다.

## 대상 후보

사용자 확인 후 다음 중 하나로 진행한다.

1. 날씨 카드 10종 전체 재생성
2. 색조가 강한 특수 날씨 5종만 재생성: 혹한, 적무, 백야, 피안, 과열
3. 색조가 강한 6종 재생성: 폭풍, 혹한, 적무, 백야, 피안, 과열

## 색 사용 기준

기본은 흑백으로 통일한다.

- 창천: 흰 하늘과 옅은 먹선
- 비: 먹 번짐과 빗줄기
- 눈: 여백과 검은 실루엣
- 안개: 회색 농담과 낮은 대비
- 폭풍: 먹물 소용돌이와 찢긴 깃발
- 혹한: 깨지는 얼음 표현은 흰 균열과 검은 윤곽 중심
- 적무: 붉은 안광만 아주 작게 허용 가능
- 백야: 창백한 흰빛과 검은 도깨비 실루엣
- 피안: 피안화는 흑백 꽃 중심, 필요하면 아주 약한 붉은 암시만 허용
- 과열: 용암은 흰 균열 또는 약한 주황빛 암시만 허용

## 구현 방향

승인 후 다음을 진행한다.

1. `task.csv`에 이번 작업 행을 추가한다.
2. 확정된 대상 카드 이미지를 새로 생성한다.
3. 기존 파일명에 맞춰 `public/card-assets/weather/`의 이미지를 교체한다.
4. `pnpm build`를 실행한다.
5. 카드 DB의 날씨 필터에서 이미지 연결과 깨짐 여부를 브라우저로 확인한다.
6. `task.csv` 상태를 `done`으로 갱신한다.

## 확인 질문

1. 날씨 카드 10종 전체를 다시 생성한다.
2. `적무`, `피안`, `과열`에 아주 약한 포인트 색을 남길지, 완전 흑백으로 갈지 확인한다.
3. 기존 파일명을 그대로 덮어쓸지, `-ink` 버전 파일로 저장하고 카드 데이터 경로를 바꿀지 확인한다.
4. 예시 이미지처럼 카드 내부의 범위 UI까지 포함한 완성 카드풍 그림을 만들지, 현재 앱의 카드 프레임에 들어갈 순수 일러스트만 만들지 확인한다.

## 대기

사용자가 `go` 또는 명확한 진행 승인을 주기 전에는 구현하지 않는다.
