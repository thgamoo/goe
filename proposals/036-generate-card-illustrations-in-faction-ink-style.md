# Proposal 036: 카드 일러스트 팩션별 수묵화 생성

## 배경

현재 온보딩 카드용 일러스트 일부가 없거나 카드별 완성도가 고르지 않다.

사용자는 `public/card-assets/illustrations/card-01.png`부터 `card-10.png`까지의 흑백 수묵화 톤처럼, 흰 여백과 검은 먹 대비가 뚜렷한 방향을 원한다. 여기에 팩션별 하이라이트 컬러를 제한적으로 더해 카드 구분감을 만든다.

## 목표

- 카드 일러스트를 5장 단위로 생성한다.
- 기본 톤은 흑백 수묵화, 먹 번짐, 한지 질감, 강한 실루엣 대비로 통일한다.
- 팩션별 하이라이트 컬러는 작은 포인트로만 사용한다.
- 카드 프레임과 텍스트가 얹히므로 이미지 안에는 글자, 로고, 테두리, 워터마크를 넣지 않는다.
- 기존 카드 아트 크기인 1024x1536 비율을 기준으로 한다.

## 팩션 컬러

- 예맥: 붉은색
- 사로국: 녹색, 옥색
- 가락: 청색
- 십제: 보라색
- 중립: 포인트 컬러 없이 흑백 중심

## 대상 후보

우선 `src/data/cards.json` 기준 온보딩 카드 전체를 대상으로 삼는다.

- `card-04`: 전차부대, 중립

### 기타 온보딩 카드

카드 번호 매핑을 승인 후 앱 렌더링 코드와 데이터 경로에서 재확인한다.

- 작전 카드 6장: 공격하라!, 퇴각하라!, 광폭, 도깨비지진, 과작동, 초혼
- 도구 카드 7장: 낙하 구덩이, 밧줄 올가미, 뾰족한 짐승덫, 맛있는 맥고기, 머리잡이, 복제화로, 무덤문
- 장군 카드 6장: 길잡의 부대의 마로, 다파나국의 왕자, 유리, 소나갈질지, 다루태자, 덕좌태자
- 대장군 카드 4장: 무신 무휼, 탈해 이사금, 구간, 내려온 건길지

## 작업 방식

승인 후 다음 순서로 진행한다.

1. `task.csv`가 없으면 생성하고, 이번 작업 행을 `in_progress`로 추가한다.
2. 카드 번호와 데이터 매핑을 코드에서 확정한다.
3. 5장 단위로 프롬프트를 작성해 수묵화 일러스트를 생성한다.
4. 생성 결과를 기존 경로 `public/card-assets/illustrations/card-XX.png`에 맞춰 저장하거나, 사용자가 원하면 별도 후보 파일로 저장한다.
5. 필요한 경우 이미지 크기를 1024x1536으로 맞춘다.
6. `pnpm build`를 실행한다.
7. 브라우저에서 카드 미리보기 화면을 확인해 이미지 깨짐, 프레임 안 잘림, 색 포인트 과다 여부를 점검한다.
8. `task.csv` 상태를 `done`으로 갱신한다.

## 확인 질문

1. 이번에 생성할 범위는 `card-01`부터 현재 존재하는 `card-27`까지인가, 아니면 카드 DB 전체의 병사/작전/도구/장군/대장군 전체인가?
2. 기존 `card-01`부터 `card-10`까지도 같은 톤으로 다시 만들까, 아니면 이 10장은 기준 이미지로 유지하고 `card-11` 이후만 만들까?
3. 생성 결과는 기존 파일명을 바로 덮어써도 될까, 아니면 `card-XX-candidate.png` 같은 후보 파일로 먼저 저장해 비교할까?
4. 카드별 장면은 카드 이름을 직관적으로 묘사하는 방향이 좋을까, 아니면 세계관 분위기를 우선해서 더 상징적이고 추상적인 방향이 좋을까?
5. 팩션 하이라이트 컬러는 눈, 무기, 문양, 배경 먹 번짐 중 어디에 가장 많이 두는 편이 좋을까?

## 대기

사용자가 `go` 또는 명확한 진행 승인을 주기 전에는 이미지 생성, 파일 교체, 빌드 같은 구현 작업을 하지 않는다.
