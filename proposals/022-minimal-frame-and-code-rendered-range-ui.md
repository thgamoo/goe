# Proposal 022: Minimal Frame And Code Rendered Range UI

## Status

Implemented.

## Context

The generated clean frame still feels too decorative for `괴이전`.

Problems:

- Gold ornamentation is too strong.
- Built-in 3x3 range slots do not fit the desired visual language.
- The card-name brush area should not be baked into the frame.
- Range direction and straight-distance indicators should be generated by code rather than baked images.

The preferred direction is to keep the frame as a quiet bitmap overlay and render variable card UI elements with React/SVG.

## Goals

- Generate or create a more minimal card frame overlay.
- Reduce gold ornamentation significantly.
- Remove baked-in 3x3 range slots.
- Remove baked-in card-title brush area.
- Keep the center illustration area transparent.
- Keep the frame mostly empty so card name, range UI, distance indicators, and other card-specific elements can be rendered above it.
- Use React/SVG for range/distance indicators, including straight-line arrows if needed.

## Non-Goals

- Do not finalize all possible range pattern rules in this pass.
- Do not generate 64 or `2^8` baked range images.
- Do not replace all card artwork.
- Do not redesign the full Card DB schema unless a small local field is needed for preview.

## Proposed Design Direction

Frame:

- Thin black ink border.
- Subtle ivory paper edge.
- No gold accent.
- No fantasy-heavy corner flourish.
- No 3x3 range sockets.
- No card-name brush strip.
- Transparent central art window.

Code-rendered UI:

- Card name and metadata remain HTML/CSS.
- Range grids and active cells should be rendered with React/SVG or CSS.
- Person marker can remain an image asset or be replaced by a simple SVG/icon later.
- Straight-distance arrows should be SVG/React, not baked raster images.

## Recommendation

Use SVG/React for arrows and distance indicators.

Reasons:

- Direction and distance vary by card.
- Scaling stays crisp in screen and print.
- Color/state can be data-driven.
- It avoids creating many baked bitmap combinations.

## User Decisions

- Follow the recommended direction: keep arrows and distance indicators code-rendered rather than baked into the frame image.
- Remove the 3x3 range slots from the frame entirely.
- Remove the card-name brush strip from the frame entirely.
- Do not add or configure `lucide-react` in this pass; the user will handle that setup later.
- Keep the person marker as PNG for now.
