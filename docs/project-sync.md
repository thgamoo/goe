# 괴이전 프로젝트 Sync

Last updated: 2026-06-15 11:26 KST

## 한 줄 요약

`괴이전`은 GRNS 세계관/IP를 차용해 만드는 보드/카드 게임 프로토타입이다. 현재는 `grns-new-game` 디렉토리에 Vite + React + shadcn/Tailwind 기반 웹사이트를 만들고, 룰/보드/카드 DB/튜토리얼/개발노트를 한 곳에서 볼 수 있게 구성 중이다.

## 저장소와 실행 환경

- 작업 디렉토리: `grns-new-game`
- 패키지 매니저: `pnpm`
- 프론트엔드: Vite, React, Tailwind CSS, shadcn/ui
- 배포 base path: `/goe/`
- 연결할 원격 저장소: `thgamoo/goe.git`
- GitHub Pages workflow가 있어 push 시 배포되도록 설정되어 있다.
- 부모 `grns-card` 저장소와 자식 `grns-new-game` 저장소는 별개로 다룬다.

## 프로젝트 작업 규칙

- 새 할 일이 생기면 먼저 `proposals/`에 proposal 문서를 만든다.
- 요구가 불명확하면 최소 3개 질문으로 사용자의 의도를 구체화한다.
- 사용자가 `go`라고 말하면 실제 작업을 시작한다.
- 작업을 시작하면 `task.csv`에 번호, 작업명, 상태, 메모, 갱신 시각을 남긴다.
- 이 규칙은 `AGENTS.md`에 저장되어 있다.

## 현재 웹사이트 구조

현재 주요 탭은 다음과 같다.

- 처음
- 룰
- 보드
- 점령카드
- 카드 DB
- 튜토리얼
- 개발노트

사용자는 랜딩 페이지의 설명성 문구를 줄이고, 실제 사용자가 보고 싶은 `룰/보드/카드 DB/튜토리얼/개발노트` 중심 구조를 원했다.

## 현재 룰 결정사항

- 게임명은 `괴이전`.
- 현재 룰 문서는 `docs/rules/v0.1.md` 하나로 합쳤다.
- 전장은 공유 5x5 보드다.
- 승리 조건은 성을 제외한 땅의 절반 이상을 점령하는 것이다.
- 성은 점령지로 세지 않는다.
- 각 플레이어는 자기 가운데 성에서 시작한다.
- 성은 5x5 전장 밖의 정사각형 칸이며, 각 플레이어 쪽 가운데 모서리 바로 바깥에 붙어 있다.
- 장군 카드는 공개하고 시작하며, 일반적으로 전투에 참여하지 않는다.
- 병사는 자신의 영향권에만 앞면으로 출정할 수 있다.
- 영향권은 자기 카드와 가로세로로 이웃한 칸이다.
- 점령은 병사가 주둔해야 유지된다.
- 점령 표시는 병사 카드를 뒤집지 않고 별도 양면 점령카드로 한다.
- 점령카드는 5x5 전장에 맞춰 25장 구성이며, 앞면은 1P, 뒷면은 2P 소유권을 나타낸다.
- 주둔을 포기하면 병사는 이동할 수 있다.
- 라운드는 `국면`, 턴은 `차례`, 액션은 `명령`이라고 부른다.
- 명령의 기본 행동은 훈련, 출정, 전투, 이동이다.
- 교전이 끝나면 국면은 무조건 전환된다.
- 작전 카드의 `힘 비용`은 교전에 참여한 병사의 `역량`을 소모해 지불한다.
- 국면마다 날씨/지형이 바뀔 수 있으나 아직 상세 규칙은 미정이다.

## 카드 구성 현황

현재 카드 DB는 `src/data/cards.json`에서 관리한다.

- 병사 카드: 15종 예시
- 장군 카드: 2종 예시
- 작전 카드: 10종 예시
- 징집소는 병사 카드 20장 구성으로 수정했다.
- 병사 카드는 같은 카드 덱에 최대 2장까지 넣을 수 있다.
- 전략은 작전 카드 20장 구성이다.
- 작전 카드는 같은 카드 덱에 최대 3장까지 넣을 수 있다.
- 장군은 플레이어당 1장이다.

## 점령카드 현황

- `점령카드` 탭을 추가했다.
- 점령카드는 병사/장군/작전 카드와 별도인 영토 표시 구성품이다.
- 목표 구성은 고유 일러스트 25장의 양면 카드다.
- 앞면은 1P, 뒷면은 2P 소유권을 나타낸다.
- 현재 예시 4종은 `src/App.tsx`의 `occupationCards` 배열에서 관리한다.
- 예시 이미지는 `public/card-assets/occupation/`에 저장되어 있다.
- 현재 예시 주제는 광소하는 광전사, 옥좌의 외뿔 도깨비, 청철 촉수거인, 보랏빛 손아귀 귀신이다.
- `점령카드` 탭은 기존 카드 DB처럼 프린트 버튼과 표준 카드 비율 미리보기를 제공한다.

## 카드 시리얼 규칙

기존 `grns-card` 카드풀의 시리얼을 기반으로 하되, 레어도를 추가한다.

예시:

- 원본: `st01-0001`
- 현재: `st01-c001`, `st01-r001`, `st01-sr001`

형식:

```text
[팩ID]-[레어도소문자][원본번호마지막세자리]
```

## 카드 프레임과 디자인 현황

- 병사 카드와 작전 카드는 기존 GRNS 카드 프레임 이미지를 사용한다.
- 장군 카드는 해당 프레임을 쓰지 않고, 스탠다드 카드 사이즈의 가로형 레이아웃을 쓴다.
- 작전 카드는 색을 빼기 위해 이미지에 `grayscale saturate-0`을 적용한다.
- 프린트 기능이 카드 DB에 있다.
- 프린트 시 카드 DB 설명 문구와 필터 UI는 숨긴다.
- 카드 크기는 프린트 기준 스탠다드 카드 비율로 고정한다.

## 카드 위치 조정 지점

카드 위치는 별도 CSS 파일이 아니라 `src/App.tsx`의 `CardPreview` 컴포넌트 안에서 Tailwind `className` 퍼센트 값으로 조정한다.

- 카드명: `src/App.tsx`의 `top-[6.9%]`, `left-[24%]`, `right-[24%]`, `h-[5.6%]`
- 좌상단 이동/힘: `left-[8.2%]`, `top-[19.6%]`, `size-[11.5%]`
- 우상단 역량/작전 표시: `right-[8.2%]`, `top-[19.6%]`, `size-[11.5%]`
- 효과 텍스트 박스: `left-[12%]`, `right-[12%]`, `top-[58.7%]`, `h-[20%]`
- 하단 진영명: `bottom-[7.8%]`, `left-[10%]`, `max-w-[31%]`
- 하단 시리얼: `bottom-[7.8%]`, `right-[10%]`, `max-w-[31%]`
- 하단 엠블럼: `bottom-[7.8%]`, `size-[12%]`

## 엠블럼 현황

- `sigil`은 카드별 상징 글자다.
- 진영 마크는 `emblem` 필드로 따로 관리한다.
- 엠블럼 이미지는 `public/card-assets/emblems/`에 복사되어 있다.
- 원본은 `../docs/faction-diamonds/*-emblem.png`에 있었다.
- 카드 하단 중앙은 이제 `sigil` 텍스트가 아니라 `emblem` 이미지를 우선 표시한다.

## 기존 카드풀 재사용 현황

기존 `grns-card` 데이터는 `../data/card-2026-06-01-v1/`를 참조했다.

- 병사/장군/작전 카드명은 기존 카드풀에서 가져온다.
- 병사 카드명은 일부 원본 카드명을 그대로 재사용한다.
- 장군 카드는 실제 `type: "성주"` 카드에서 골랐다.
- 현재 장군 예시는 `하룻 이리`, `평화주의 곰`이다.
- 작전 카드는 “작전처럼 보이는 이름”만 기존 카드풀에서 골라 다시 배치했다.

## 현재 작전 카드명

작전 카드명은 병사/유닛처럼 보이는 이름을 피하고, 행동/명령/전술/교전 상황처럼 읽히는 이름을 골랐다.

현재 10종이며, 이름에 맞춰 효과를 1차 재작성했다:

- 그림자 함정
- 북천 돌파령
- 철갑 전열
- 퇴각하라
- 왕명 사칭
- 쇠사슬 봉기
- 도깨비 몰락 방울
- 좌우전이
- 분노의 역습
- 패태자의 전략

역할은 급습 방해, 증원, 방어, 퇴각, 작전 방해, 묶기, 방어 정리, 역량 교환, 역습, 항복 보상으로 나뉜다.

## 미해결 쟁점

- 카드 프레임 위 텍스트 위치가 아직 완전히 맞지 않는다.
- 사용자가 직접 좌표를 조정할 수 있도록 `CardPreview` 위치값을 안내했다.
- 장군이 전투에 참여하는 예외 조건은 아직 미정이다.
- 장군 능력과 추가 행동 처리 순서는 아직 `그 외`로 둔다.
- 날씨/지형 국면 전환 규칙은 빈 섹션만 있고 상세화 전이다.
- 보드 화면은 플레이어 쪽 2.5x5 전장과 가운데 모서리 밖의 성 칸을 보여주는 방향으로 수정했다.

## 다른 LLM에게 전달할 말

이 프로젝트는 아직 룰과 카드가 빠르게 바뀌는 초기 프로토타입이다. 기존 `grns-card`의 데이터와 에셋을 재사용하지만, `괴이전`은 독립 repo인 `grns-new-game`에서 진행한다. 새 변경을 만들 때는 먼저 `proposals/`에 제안서를 만들고, 사용자의 `go`를 받은 뒤 `task.csv`에 작업을 추가하고 구현한다.
