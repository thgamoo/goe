# 괴이전 프로젝트 Sync

Last updated: 2026-06-28 13:50 KST

## 한 줄 요약

`괴이전`은 GRNS 세계관/IP를 차용해 만드는 보드/카드 게임 프로토타입이다. 현재는 `grns-new-game` 디렉토리에 Vite + React + shadcn/Tailwind 기반 웹사이트를 만들고, 룰/보드/카드 DB/튜토리얼/개발노트를 한 곳에서 볼 수 있게 구성 중이다.

## 저장소와 실행 환경

- 작업 디렉토리: `grns-new-game`
- 패키지 매니저: `pnpm`
- 프론트엔드: Vite, React, Tailwind CSS, shadcn/ui
- 배포 base path: `/goe/`
- 연결할 원격 저장소: `thgamoo/goe.git`
- GitHub Pages workflow가 있어 push 시 배포되도록 설정되어 있다.
- 부모 `grns-card` 저장소와 자식 `grns-new-game` 저장소는 별개로 다룬다.

## 프로젝트 작업 규칙

- 새 할 일이 생기면 먼저 `proposals/`에 proposal 문서를 만든다.
- 요구가 불명확하면 최소 3개 질문으로 사용자의 의도를 구체화한다.
- 사용자가 `go`라고 말하면 실제 작업을 시작한다.
- 작업을 시작하면 `task.csv`에 번호, 작업명, 상태, 메모, 갱신 시각을 남긴다.
- 이 규칙은 `AGENTS.md`에 저장되어 있다.

## 현재 웹사이트 구조

현재 주요 탭은 다음과 같다.

- 처음
- 룰
- 보드
- 점령카드
- 카드 DB
- 튜토리얼
- 개발노트

사용자는 랜딩 페이지의 설명성 문구를 줄이고, 실제 사용자가 보고 싶은 `룰/보드/카드 DB/튜토리얼/개발노트` 중심 구조를 원했다.

## 현재 룰 결정사항

- 게임명은 `괴이전`.
- 현재 룰 문서는 `docs/rules/v0.1.md` 하나로 합쳤다.
- 전장은 공유 5x5 보드다.
- 승리 조건은 성을 제외한 땅의 절반 이상을 점령하는 것이다.
- 성은 점령지로 세지 않는다.
- 각 플레이어는 자기 가운데 성에서 시작한다.
- 성은 5x5 전장 밖의 정사각형 칸이며, 각 플레이어 쪽 가운데 모서리 바로 바깥에 붙어 있다.
- 대장군 카드는 공개하고 시작하는 리더 카드다.
- 대장군은 필드에 일반적으로 나올 수 없고, 자기 차례에 한 번 `명령`을 사용할 수 있다.
- 기존 장군 카드는 기지에 들어가는 세로형 카드다.
- 병사는 자신의 영향권에만 앞면으로 출정할 수 있다.
- 병사는 낸 차례에 바로 이동하거나 공격할 수 없다.
- 영향권은 자기 카드와 가로세로로 이웃한 칸이다.
- 점령은 병사가 주둔해야 유지된다.
- 점령 표시는 병사 카드를 뒤집지 않고 별도 양면 점령카드로 한다.
- 점령카드는 5x5 전장에 맞춰 25장 구성이며, 앞면은 1P, 뒷면은 2P 소유권을 나타낸다.
- 주둔을 포기하면 병사는 이동할 수 있다.
- 라운드는 `국면`, 턴은 `차례`라고 부른다.
- `명령`은 대장군 전용 용어다.
- 기본 차례 행동은 보급, 출정, 전투, 이동이다.
- `전투`는 하나의 유닛과 유닛 사이에서 일어나는 한 번의 싸움이다.
- `교전`은 전투 중인 병사 또는 장군들이 상대의 공격범위에 엮여 만들어진 클러스터다.
- 교전은 페이즈가 아니라 전장 상태다.
- 교전이 끝나면 국면은 전환된다.
- 국면 전환 시 날씨가 바뀐다.
- 기지는 병사, 장군, 도구, 작전 문서가 함께 들어 있는 보급 카드 묶음이다.
- 차례 시작마다 기지에서 4장을 보급받는다.
- 사용하지 않을 카드는 제한 없이 보류로 뒤집어 놓을 수 있다.
- 다음 차례에는 보류 카드 수만큼 추가 보급을 받는다.
- 보류 카드는 상대에게 장수만 공개한다.
- 보류 카드는 추가 보급 후 원하는 순서로 기지 아래에 놓는다.
- 작전 카드의 정식 명칭은 `작전 문서`이며, 보통 `작전`이라고 부른다.
- 반응 작전은 공격 선언 후 또는 상대 작전 선언 후 사용할 수 있다.
- 모든 카드의 소모/생존/점령 단위 표현은 `병력`으로 통일한다.
- 사용된 카드와 전사한 카드는 플레이어별 `매장지`로 간다.
- `[단말마]`는 카드가 전사했을 때 발동한다.
- `[자폭]`은 공격범위 내 상대 병사가 있을 때 자신과 상대 병사를 전사시키는 키워드다.
- `귀신`은 종족이다.
- 도구는 병사에게 장착하거나 필드 위 장악범위 내 칸에 배치한다.
- 도구가 놓인 칸에는 점령카드를 함께 놓을 수 없지만, 병사 카드는 함께 놓일 수 있다.
- 초혼으로 배치한 병력 토큰은 이번 차례 동안 장악만 할 수 있고 차례가 끝나면 사라진다.
- 대장군 4종은 온보딩 문서에 추가되어 있다: 무신 무휼, 탈해 이사금, 구간, 내려온 건길지.

## 카드 구성 현황

현재 카드 DB는 `src/data/cards.json`에서 관리한다.

- 카드 DB는 온보딩 카드풀 중심으로 재구성되어 있다.
- 병사 카드: 15종
- 장군 카드: 6종
- 작전 문서: 6종
- 도구: 7종
- 날씨: 10종. 날씨는 온보딩 덱 40장 계산에서 제외하지만 DB에는 유지한다.
- 대장군: 4종. 카드 DB의 별도 `대장군` 서브탭에서 확인하고 프린트한다.
- 기지는 공통 카드 30장 + 진영/클래스 특성 카드 10장 = 40장 구성이다.
- 공통 카드는 중립 병사 15장, 중립 도구 9장, 중립 작전 문서 6장이다.
- 예맥, 사로국, 가락, 십제는 각각 특성 병사/장군 6장과 특성 도구/작전 문서 4장을 더해 40장 덱이 된다.
- 병사 카드는 같은 카드 기지에 최대 2장까지 넣을 수 있다.
- 작전 문서는 같은 카드 기지에 최대 3장까지 넣을 수 있다.
- 도구 카드의 장수 제한은 아직 미정이다.
- 대장군은 플레이어당 1장이다.
- 온보딩 카드별 원천 문서는 `docs/cards/onboarding/`에 있다.
- `병력`은 비용이 아니다. 작전 문서와 도구는 비용 표시를 하지 않는다.
- 카드 타입은 `병사`, `도구`, `작전 문서`, `장군`, `날씨`, `대장군`이다.
- 카드 종족은 `인간`, `인간/짐승`, `철기`, `귀신`, `도깨비`처럼 별도 필드로 관리한다.
- 왼쪽 아래 표기는 `타입 · 종족` 형식이다. 도구/작전 문서/날씨는 종족 없이 타입만 표기한다.
- 진영의 공용 표기는 `공통`이 아니라 `중립`을 쓴다.
- 날씨 카드는 진영을 갖지 않고, 왼쪽 아래에는 `날씨`만 표시한다.
- 카드 DB 키워드 필터는 현재 룰 키워드인 `출정`, `매복`, `보류`, `자폭`, `단말마`, `전투광`, `명령`만 노출한다.

## 점령카드 현황

- `점령카드` 탭을 추가했다.
- 점령카드는 병사/장군/도구/작전 문서와 별도인 영토 표시 구성품이다.
- 목표 구성은 고유 일러스트 25장의 양면 카드다.
- 앞면은 1P, 뒷면은 2P 소유권을 나타낸다.
- 현재 예시 4종은 `src/App.tsx`의 `occupationCards` 배열에서 관리한다.
- 예시 이미지는 `public/card-assets/occupation/`에 저장되어 있다.
- 현재 예시 주제는 광소하는 광전사, 옥좌의 외뿔 도깨비, 청철 촉수거인, 보랏빛 손아귀 귀신이다.
- `점령카드` 탭은 표준 카드 비율 미리보기와 앞면/뒷면 분리 프린트 기능을 함께 제공한다.

## 프린트 현황

- 별도 상단 `프린트` 탭은 제거했다.
- 점령카드 프린트는 `점령카드` 탭 안에서 처리한다.
- 대장군 프린트는 카드 DB의 `대장군` 서브탭에서 필터링된 카드만 출력하는 방식으로 처리한다.
- 점령카드 프린트는 기존 4종 점령카드를 종류별로 27장씩 만든다.
- 앞면과 뒷면은 따로 출력한다.
- 카드 1장의 출력 치수는 `63mm x 88mm`다.
- 한 페이지에는 3x3, 총 9장을 간격 없이 배치한다.
- 총 108장 구성이라 앞면 12페이지, 뒷면 12페이지가 된다.
- 각 종류는 27장씩이며, 종류별로 3페이지를 꽉 채운다.
- 화면 미리보기에는 이름과 번호가 보이지만, 실제 프린트 면에는 이미지만 출력한다.

## 카드 시리얼 규칙

기존 `grns-card` 카드풀의 시리얼을 기반으로 하되, 레어도를 추가한다.

예시:

- 원본: `st01-0001`
- 현재: `st01-c001`, `st01-r001`, `st01-sr001`

형식:

```text
[팩ID]-[레어도소문자][원본번호마지막세자리]
```

## 카드 프레임과 디자인 현황

- 병사 카드, 장군 카드, 작전 문서, 도구는 현재 `public/card-assets/common/goe-card-frame-minimal-v1.png`를 카드 DB 미리보기 프레임으로 사용한다.
- `goe-card-frame-minimal-v1.png`는 카드 스탠다드 비율에 맞춘 투명 PNG이며, 금장식/3x3 범위 슬롯/카드명 붓칠 라벨을 제거한 흑묵+종이 질감 프레임이다.
- 카드명, 범위, 직선거리 화살표 같은 카드별 UI는 프레임 이미지에 굽지 않고 React/CSS/SVG 레이어에서 올리는 방향이다.
- 카드명은 `public/card-assets/common/goe-card-brush-label-v1.png`를 `CardTitleLabel` 컴포넌트에서 사용한다.
- 병력 검은 원, 종족 밑줄, 레어도/시리얼 표기는 PNG가 아니라 React/CSS 컴포넌트다.
- 하단 중앙은 한자 `sigil` 대신 엠블럼을 표시한다.
- 공통/중립 및 날씨 엠블럼은 `public/card-assets/emblems/tu01-emblem.png`를 사용한다.
- 새 카드 프레임 레퍼런스는 `public/card-assets/reference/card-design.webp`에 보관한다.
- 해당 레퍼런스에서 추출한 프레임 초안은 `public/card-assets/common/goe-card-frame-v2.png`다.
- `goe-card-frame-v2.png`는 중앙 일러스트 영역과 상단 범위 예시 영역을 투명하게 비운 PNG다.
- 더 깨끗한 비교본은 `public/card-assets/common/goe-card-frame-v3.png`다.
- `goe-card-frame-v3.png`는 중앙 일러스트, 상단 범위 UI, 좌우 대나무/먹 번짐 꼬투리를 더 넓게 제거하고 외곽/코너/하단 UI 중심으로 남긴 PNG다.
- 이전 클린 프레임 비교본은 `public/card-assets/common/goe-card-frame-clean-v1.png`다.
- `goe-card-frame-clean-v1.png`는 카드 스탠다드 비율에 맞춘 투명 PNG이며, 중앙 일러스트/상단 범위/하단 라벨 영역을 비워둔 오버레이다.
- 별도 하단 붓터치 라벨은 `public/card-assets/common/goe-card-brush-label-v1.png`다.
- 범위 중앙에 올릴 사람 마커 아이콘은 `public/card-assets/ranges/range-person-marker-v1.png`다.
- 범위 표시는 조합 이미지를 미리 굽지 않고, 재사용 가능한 투명 PNG 조각으로 관리한다.
- 범위 격자 프레임은 `public/card-assets/ranges/range-grid-frame.png`다.
- 장악범위 활성 칸은 `public/card-assets/ranges/range-cell-control.png`다.
- 공격범위 활성 칸은 `public/card-assets/ranges/range-cell-attack.png`다.
- 일반 장군 카드는 병사와 같은 세로형 스탠다드 카드 레이아웃을 쓴다. 가로형 레이아웃은 별도 `대장군` 카드용으로 새로 만든다.
- 작전 문서는 색을 빼기 위해 이미지에 `grayscale saturate-0`을 적용한다.
- 프린트 기능은 `점령카드` 탭과 카드 DB의 현재 필터 결과 출력으로 나뉜다.
- 프린트 시 카드 DB 설명 문구와 필터 UI는 숨긴다.
- 카드 크기는 프린트 기준 스탠다드 카드 비율로 고정한다.

## 카드 위치 조정 지점

카드 위치는 별도 CSS 파일이 아니라 `src/App.tsx`의 `CardPreview` 컴포넌트 안에서 Tailwind `className` 퍼센트 값으로 조정한다.

- 카드명 붓 라벨: `CardTitleLabel`, `left-[20%]`, `right-[20%]`, `top-[4.9%]`, `h-[8.4%]`
- 장악범위 3x3 그리드: `RangeGrid`, `left-[9.8%]`, `top-[13.8%]`, `w-[18.5%]`
- 공격범위 3x3 그리드: `RangeGrid`, `right-[9.8%]`, `top-[13.8%]`, `w-[18.5%]`
- 병력 검은 원: `PowerCircle`, `right-[12%]`, `top-[31.7%]`, `size-[12.5%]`
- 일러스트: `left-[11.3%]`, `top-[9.5%]`, `w-[77.5%]`, `h-[66.3%]`
- 효과 텍스트 박스: `left-[12%]`, `right-[12%]`, `top-[76.8%]`, `h-[8.8%]`
- 하단 종족/타입: `SpeciesLine`, `bottom-[7.3%]`, `left-[9.5%]`
- 하단 레어도/시리얼: `CardMeta`, `bottom-[7.3%]`, `right-[9.5%]`
- 하단 엠블럼: `bottom-[4.2%]`, `size-[10%]`

## 엠블럼 현황

- `sigil`은 카드별 상징 글자다.
- 진영 마크는 `emblem` 필드로 따로 관리한다.
- 엠블럼 이미지는 `public/card-assets/emblems/`에 복사되어 있다.
- 원본은 `../docs/faction-diamonds/*-emblem.png`에 있었다.
- 카드 하단 중앙은 이제 `sigil` 텍스트가 아니라 `emblem` 이미지를 우선 표시한다.

## 기존 카드풀 재사용 현황

기존 `grns-card` 기반 예시 데이터는 Card DB에서 제거했다. 현재 Card DB의 병사/장군/작전 문서/도구는 온보딩 문서 기반이다.

## 미해결 쟁점

- 카드 프레임 위 텍스트 위치가 아직 완전히 맞지 않는다.
- 사용자가 직접 좌표를 조정할 수 있도록 `CardPreview` 위치값을 안내했다.
- 장군 카드의 기지 투입 제한은 아직 미정이다.
- 카드 능력과 추가 행동 처리 순서는 아직 `그 외`로 둔다.
- 날씨/지형 국면 전환 규칙은 방향만 있고 상세화 전이다.
- 날씨 카드는 각 플레이어가 3장씩 고르는 방향이다.
- 보드 화면은 플레이어 쪽 2.5x5 전장과 가운데 모서리 밖의 성 칸을 보여주는 방향으로 수정했다.

## 다른 LLM에게 전달할 말

이 프로젝트는 아직 룰과 카드가 빠르게 바뀌는 초기 프로토타입이다. 기존 `grns-card`의 데이터와 에셋을 재사용하지만, `괴이전`은 독립 repo인 `grns-new-game`에서 진행한다. 새 변경을 만들 때는 먼저 `proposals/`에 제안서를 만들고, 사용자의 `go`를 받은 뒤 `task.csv`에 작업을 추가하고 구현한다.
